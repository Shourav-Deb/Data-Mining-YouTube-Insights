INPUT

Bring ideas down to realistic 4-month, 10–12 mark lab project scope. 
ROLE AND GOAL
You are assisting an undergraduate CS student to choose a REALISTIC MINI-PROJECT based on ONE CS paper.

INPUT
I will give you: Method description, Extension Ideas List.
CONSTRAINTS:
- 4 months duration.
- Team of 3 students.
- Limited compute (Laptop/Colab Free Tier).

STRICT RULES
- Respect the constraints. Do NOT assume large clusters or huge datasets.
- It is okay if the project is modest; it should be non-trivial but realistic.

TASK
For EACH candidate idea:
1. Feasibility Analysis (yes/no/maybe)
2. Minimal Viable Version (MVV) with 3–5 milestones.
3. Risks.

OUTPUT FORMAT
- Idea: <title>
  Feasibility: ...
  Minimal Viable Version: ...
  Risks: ...

P10 – Report Skeleton Generator

👨‍🏫 Teacher's Goal
Give them a structured report outline that matches your staircase (mental model → critique → mini-project). 
ROLE AND GOAL
You are assisting an undergraduate CS student to design a REPORT OUTLINE for a small research-style lab project.

INPUT
I will give you: Paper Title, Summary, Component Map, Exp Matrix, Assumptions, Chosen Mini-Project Idea.

STRICT RULES
- clearly separate: What the original paper did vs. What we (the students) did.
- Do NOT fill in content; just give headings and bullet points.

TASK
Propose a detailed report skeleton. Mandatory parts:
1. Introduction
2. Background
3. Understanding Original Method
4. Critical Analysis
5. Proposed Mini-Project
6. Implementation (Our Work)
7. Results (Our Work)
8. Conclusion

OUTPUT FORMAT
- Section Heading
  - Purpose:
  - Students should write: [bullets]
